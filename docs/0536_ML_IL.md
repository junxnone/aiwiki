# 增量学习/持续学习

- Incremental Learning - Lifelong Learning - Continuous Learning - Never Ending Learning
- 新增数据时，只做关于新增数据的更新，增量学习可以不断从新样本中学习新的知识，同时保存大部分以前学习到的知识
  - **知识保留（Stability）**：旧任务性能不骤降，抑制灾难性遗忘。
  - **知识迁移（Plasticity）**：用旧知识加速新任务学习，提升泛化。
  - **资源高效**：有限算力/存储下适配动态任务扩展。
- **挑战**
  - **灾难性遗忘（Catastrophic Forgetting）**：新任务梯度覆盖旧任务参数，旧性能断崖式下跌。
  - **负迁移**：旧知识干扰新任务学习，性能不升反降。
  - **无监督 / 弱监督适配**：工业几乎无充足异常标注，CL 需在仅正常样本下实现
  - **任务边界模糊**：真实场景无明确任务划分，需在线/无界学习。
  - **资源约束**：边缘/低算力设备无法全量微调与大回放。
  - **检测精度与稳定性**：工业要求零漏检、低误检，CL 更新必须可控、可回滚

## 任务分类

设定 | 描述 | 典型场景
-- | -- | --
类别增量（Class-IL） | 逐次新增类别，测试需区分所有新旧类 | 工业缺陷检测、图像分类
领域增量（Domain-IL） | 类别不变，数据分布 / 域变化 | 跨设备 / 跨光照异常检测
任务增量（Task-IL） | 任务独立，需指定任务 ID 推理 | 多任务 NLP、多模态指令微调
通用持续学习（GCL） | 无任务边界、无任务 ID、数据流连续 | 在线推荐、实时视觉感知


> - 产线生产细微差异产品 ==> 类别增量
> - 产线调整迁移 ==> 领域增量


## 主流方法
### 1. 正则化方法（Regularization）
- **核心思想**：在更新新任务时，对旧任务重要参数施加约束，减少遗忘
- **工业适配**：
  - **EWC（弹性权重 consolidation）**：计算参数重要性，限制关键参数更新
  - **LwF（Learning without Forgetting）**：用旧模型输出做知识蒸馏，保留旧知识
  - **工业改进**：结合异常检测的**正常样本建模**特性，仅约束正常特征空间
- **代表工作**：AETS（适配器增强双阶段持续缺陷检测），冻结主干+适配器微调，缓解遗忘

### 2. 记忆回放（Memory Replay）
- **核心思想**：存储少量历史样本/特征，学习新任务时混合回放，模拟旧数据分布
- **工业适配（主流路线）**：
  - **特征记忆库**：PatchCore等基于特征库的方法，天然支持增量更新
  - **核心集（Coreset）选择**：k-center、代表性采样，仅存最具区分度的正常特征
  - **无监督回放**：仅回放正常样本，避免异常稀缺问题
- **代表工作**：
  - **On-Device CL for PatchCore**：轻量主干+增量核心集，内存降80%、AUROC+12%
  - **UCAD（AAAI 2024）**：对比学习提示+记忆库，无监督持续检测+分割

### 3. 参数隔离（Parameter Isolation）
- **核心思想**：为不同任务分配独立参数/模块，互不干扰
- **工业适配**：
  - **适配器（Adapter）**：在预训练模型中插入轻量适配器，冻结主干，仅训练适配器
  - **专家混合（MoE）**：动态路由不同任务到对应专家模块
  - **提示学习（Prompt Learning）**：L2P、任务提示，用少量可学习提示编码任务信息
- **代表工作**：AETS、UCAD（对比提示）、基于Adapter的ViT持续检测

### 4. 动态架构（Dynamic Architecture）
- **核心思想**：模型随任务增长动态扩展，不覆盖旧参数
- **工业适配**：
  - **渐进式网络**：新增任务时增加分支/层
  - **动态特征库**：PatchCore的记忆库动态扩容
  - **持续扩散模型**：One-for-More（CVPR 2025），用扩散模型持续生成正常模式


### 方法对比（工业IAD场景）

| 方法 | 优点 | 缺点 | 工业适用性 |
|:--- |:--- |:--- |:--- |
| 正则化 | 计算轻、无额外存储 | 遗忘缓解有限、难适配强漂移 | 低变化场景 |
| 记忆回放 | 遗忘抑制强、适配无监督 | 需管理记忆库、存在隐私风险 | **主流首选** |
| 参数隔离 | 遗忘少、适配多任务 | 参数量随任务增长、推理略慢 | 多产品线场景 |
| 动态架构 | 理论最优、无遗忘 | 复杂、部署难、算力要求高 | 前沿探索 |


## 技术路线

### 1. 无监督持续异常检测（主流）
- **核心设定**：仅用正常样本，顺序学习多类产品/环境
- **技术栈**：
  - **特征提取**：轻量CNN（MobileNet）、ViT、CLIP特征
  - **持续模块**：增量特征库、适配器、提示学习
  - **异常评分**：距离度量（欧氏、马氏）、重构误差、似然估计
- **代表框架**：
  - **UCAD**：对比提示+记忆库，支持检测+分割
  - **增量PatchCore**：核心集更新+k-center采样，边缘端部署

### 2. 弱监督持续异常检测
- **核心设定**：少量图像级异常标签，无像素级标注
- **技术栈**：
  - 持续学习+弱监督定位（CAM、注意力）
  - 伪异常生成+持续蒸馏
- **代表工作**：IAD-R1（ICCV 2025），强化持续推理一致性

### 3. 全监督持续异常检测
- **核心设定**：充足异常标注，适配缺陷分类+定位
- **技术栈**：持续目标检测/分割（Faster R-CNN、Mask R-CNN变体）+ CL策略
- **应用**：缺陷类型明确、标注充分的产线
