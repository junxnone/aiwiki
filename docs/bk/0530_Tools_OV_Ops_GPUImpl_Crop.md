# GPU Impl Crop

  - 从输入张量（如 bfyx/b\_fs\_yx\_fsv16 格式）中，按照起始坐标 +
    截取尺寸提取子张量，支持多维度（批次、通道、高度、宽度、深度）的裁剪，且完全适配
    GPU 的阻塞格式（无需先转回线性格式）。
  - 输入格式：支持 bfyx（线性）、b\_fs\_yx\_fsv16/b\_fs\_yx\_fsv32（GPU
    阻塞格式）、nhwc（跨框架适配）等；
  - 输出格式：默认与输入格式一致（如输入 b\_fs\_yx\_fsv16，输出也为该格式），避免额外的 reorder 开销；
  - 特殊优化：对阻塞格式的裁剪无需拆分组结构，直接在分组内截取，保证 GPU 访存效率。

| 维度缩写 | 含义     | 是否支持裁剪 | 典型场景            |
| ---- | ------ | ------ | --------------- |
| b    | 批次     | 是      | 截取批次中的部分样本      |
| f    | 通道     | 是      | 截取特征图的部分通道      |
| y    | 高度（空间） | 是      | 裁剪图像 / 特征图的高度区域 |
| x    | 宽度（空间） | 是      | 裁剪图像 / 特征图的宽度区域 |
| z    | 深度（3D） | 是      | 3D 医学影像的深度裁剪    |

## crop Kernel 的典型应用场景

  - \*\*目标检测的 ROI 裁剪：\*\*在目标检测模型（如 YOLO、Faster
    R-CNN）中，截取候选框（ROI）对应的特征图区域，供后续分类
    / 回归计算；
  - **图像局部区域分析**：如只处理图像的中心区域、去除边缘无效区域；
  - **特征图维度适配**：裁剪特征图到指定尺寸，满足下游算子的输入维度要求；
  - **3D 数据处理**：对医学影像（如 CT/MRI 的 3D 张量）进行深度维度的裁剪，提取感兴趣的体素区域。

## 关键优化与注意事项

  - **格式对齐优化**：若输入为 GPU 阻塞格式（如 b\_fs\_yx\_fsv16），crop Kernel
    会直接在分组内截取，避免先转为 bfyx 再裁剪（减少内存拷贝）；
  - **越界保护**：crop Kernel 内置越界检查，若 offsets + dimensions
    超过输入维度，会自动截断（或抛出明确的越界异常）；
  - **融合优化**：在图优化阶段，crop 可能与相邻算子（如 reorder、conv）融合，例如 “裁剪 + 格式转换”
    一次性完成，减少 Kernel 调用次数；
  - **性能建议**：裁剪维度尽量对齐 GPU 缓存行（如裁剪宽度 / 高度为 16 的倍数），可进一步提升访存效率。

## 与 pytorch 的对应关系

| OpenVINO crop 能力            | PyTorch 等价算子 / 组合方式                         | 适用场景           |
| --------------------------- | ------------------------------------------- | -------------- |
| 基础维度裁剪（指定起始 + 尺寸）           | torch.narrow() + torch.index\_select()（低维度） | 单维度 / 少维度裁剪    |
| 任意多维度裁剪（通用场景）               | tensor\[切片索引\]（Python 原生切片语法）               | 绝大多数裁剪场景（推荐）   |
| 动态 / 编程式裁剪（指定 offsets/size） | torch.slice()（PyTorch 2.0+）                 | 需动态传入起始 / 尺寸参数 |
| 高级裁剪（含维度对齐 / 补零）            | torch.nn.functional.pad() + 切片（反向裁剪）        | 需补零后再裁剪的场景     |
